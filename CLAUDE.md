# CLAUDE.md — セキュリティ規約 & 開発ガイドライン

## このファイルについて
Claude Code がこのプロジェクトで作業する際に必ず遵守するルールです。
違反しようとする実装を検知した場合は、**必ずユーザーに確認を求めること。**

---

## セキュリティ規約（必須遵守）

### 環境変数
- 環境変数は `.env.local` にのみ記載する
- `.env.local` は絶対にコミットしない（.gitignore で除外済み）
- `.env.example` に変数名のみ記載し、実値は空にする
- コード内に API キー・認証情報・シークレットを直書きしない

### 自分で更新
- 出力前に検算する
- 推測で答えない
- ファイル変更は差分単位で報告
- 完了報告を鵜呑みにしない側として書く
- 不明点は聞く

### 入力バリデーション
- ユーザー入力は必ず **Zod** でバリデーションしてから使用する
- バリデーション前のデータを DB・外部 API に渡さない

### ログ・エラー
- ログ出力前に PII（メールアドレス・トークン・氏名・電話番号等）を必ずマスキングする
  - 例: `user@example.com` → `u***@example.com`
- ユーザー向けエラーメッセージにスタックトレース・内部情報を含めない
  - OK: `処理に失敗しました。しばらくしてから再試行してください。`
  - NG: `Error: DB connection failed at /src/lib/db.ts:42`

### 外部 API
- 外部 API の使用は原則禁止
- 使用が必要な場合は事前にユーザーへ確認・承認を得ること

### 違反チェックリスト（実装前に確認）
- [ ] ハードコードされた認証情報がないか
- [ ] ユーザー入力に Zod バリデーションがあるか
- [ ] ログに PII が含まれていないか
- [ ] エラーメッセージが安全か
- [ ] 外部通信が発生しないか

## プロジェクト構成

```
src/
  app/          # App Router ページ・レイアウト
  components/   # 再利用可能コンポーネント
  lib/          # ユーティリティ・ヘルパー
  types/        # TypeScript 型定義
```

## 静的エクスポートの制約
`output: 'export'` を使用しているため以下は使用不可:
- `next/headers` (cookies/headers)
- Route Handlers でのリクエスト依存処理
- Image Optimization (`unoptimized: true` を使用)
